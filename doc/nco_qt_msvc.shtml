NCO Qt/MSVC build

This page contains instructions regarding building NCO with the Qt system and with Microsoft Windows Visual Studio 2010 (MSVC). This page is mostly important for developers that want to build NCO from source using Qt or MSVC.

Build NCO with Qt
Qt is a cross platform build system. As of NCO 4.2.1, it is possible to build NCO with Qt, in a Linux, Mac OSX or Windows enviroment. NCO can be built either from the Qt IDE, QtCreator, or from the command line. To build NCO with QtCreator, just open the file /qt/nco.pro. To build Qt from the command line, do:

$ cd qt
$ qmake
$ make

NCO depends on several pre-installed libraries, namely HDF5 (including HDF5 High-Level), netCDF, zlib, UDunits and Expat. The NCO supplied Qt projects assume static builds of these libraries in the following locations:

/usr/local/lib/libnetcdf.a
/usr/local/lib/libhdf5_hl.a
/usr/lib/x86_64-linux-gnu/libz.a
/usr/lib/x86_64-linux-gnu/libudunits2.a
/usr/lib/x86_64-linux-gnu/libexpat.a
/usr/lib/x86_64-linux-gnu/libcurl.a

Build NCO with MSVC
As of NCO 4.2.0, it is possible to build NCO with Visual Studio 2010. To build NCO with MSVC, open the file /qt/nco.sln with Visual Studio.

NCO depends on several pre-installed libraries, namely HDF5 (including HDF5 High-Level), netCDF, zlib, UDunits and Expat. The NCO supplied Visual Studio projects assume static builds of these libraries. These are specified by means of the following Windows enviroments variables in the projects:

LIB_NETCDF
LIB_DISPATCH
LIB_NETCDF4
LIB_HDF5
LIB_HDF5_HL
LIB_ZLIB
LIB_SZIP
LIB_UDUNITS
LIB_EXPAT

Additionnaly, the following environment variable must also be defined for the location of the netCDF header files:

HEADER_NETCDF
